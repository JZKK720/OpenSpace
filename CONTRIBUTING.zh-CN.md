# 贡献指南

感谢你的贡献。

本文件为中文便捷版本。如与英文原文存在任何冲突，以 [CONTRIBUTING.md](CONTRIBUTING.md) 为准。

## 代码贡献

除非另有书面约定，对本仓库的代码、文档、测试、配置与其他非品牌内容的贡献，均按 [LICENSE](LICENSE) 中的 MIT 协议接收。

提交贡献即表示你同意这些内容可以按照该 MIT 协议被使用、修改和再分发。

## 品牌与商标边界

向本仓库贡献内容，并不会授予你以下任何权利：

- `Cubecloud` 名称的使用权
- `cubecloud.io` 域名的使用权
- Cubecloud 的 logo、图标、字标、徽章或整体品牌呈现的使用权
- [BRAND_ASSETS.zh-CN.md](BRAND_ASSETS.zh-CN.md) 中列明的其他 Cubecloud 品牌资产

这些权利仍由 [TRADEMARKS.zh-CN.md](TRADEMARKS.zh-CN.md) 保留。

## Fork 与公开发布

如果你公开发布一个 fork，建议你：

- 保留上游署名与许可声明
- 若未取得 Cubecloud 品牌书面许可，请为公开产品使用新的名称
- 若未取得书面许可，请替换用户界面中的 Cubecloud logo 与品牌标识
- 不要将你的 fork 描述为官方 Cubecloud 构建

## Pull Request

请尽量让 Pull Request 保持技术范围清晰，并说明：

- 改了什么
- 为什么改
- 如何验证

如果变更涉及品牌、logo、命名或公共身份，应将其视为需要维护者审核的品牌变更，而不仅仅是普通代码改动。
